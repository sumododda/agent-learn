# PDF Course Export Design

**Date:** 2026-03-29
**Status:** Draft

## Overview

Add the ability to export a completed course as a nicely formatted PDF with a classic textbook style. The PDF is generated server-side using Typst (via `typst-py`) and served as a file download.

## Scope

**In scope:**
- Backend PDF generation endpoint
- Typst template with classic textbook styling (serif fonts, clean margins, footnotes)
- Cover page with course title and generation date
- All section content rendered from markdown
- References/bibliography section at the end with all cited sources
- "Export PDF" button on the learning page header

**Out of scope:**
- Evidence cards, glossary, or blackboard content in the PDF
- Frontend (client-side) PDF generation
- Export to other formats (EPUB, HTML, DOCX)
- Customizable styling or theme options

## Architecture

### Data Flow

```
User clicks "Export PDF" on learning page
  → Frontend: GET /api/courses/{id}/export/pdf (with auth token)
  → Backend: Fetch course + sections + citations from DB
  → Backend: Convert section markdown to Typst markup
  → Backend: Render Typst template with course data
  → Backend: typst-py compiles Typst → PDF bytes
  → Backend: Return Response(content=pdf_bytes, media_type="application/pdf")
  → Browser: Downloads file as "{course_topic}.pdf"
```

### Backend

#### New Endpoint

```
GET /api/courses/{id}/export/pdf
```

- **Auth:** Requires valid JWT. User must own the course.
- **Validation:** Course must have status `completed`. Returns 400 if not.
- **Response:** `application/pdf` with `Content-Disposition: attachment; filename="{sanitized_topic}.pdf"`
- **Errors:** 404 if course not found, 403 if not owner, 400 if not completed.

#### New Module: `backend/app/pdf_export.py`

Responsible for:
1. Accepting a `CourseResponse` (with sections and citations)
2. Converting markdown content to Typst markup
3. Populating the Typst template
4. Compiling to PDF via `typst-py`
5. Returning PDF bytes

#### Markdown-to-Typst Conversion

Course section content is stored as markdown. Typst has its own markup syntax. The conversion handles:

| Markdown | Typst |
|----------|-------|
| `# Heading` | `= Heading` |
| `## Heading` | `== Heading` |
| `**bold**` | `*bold*` |
| `*italic*` | `_italic_` |
| `[text](url)` | `#link("url")[text]` |
| `` `code` `` | `` `code` `` (same) |
| ```` ```lang ... ``` ```` | `` ```lang ... ``` `` (same) |
| `- item` | `- item` (same) |
| `1. item` | `+ item` |
| `> blockquote` | `#quote[...]` |
| `![alt](url)` | `#image("url", alt: "alt")` |
| `[1]` (citation ref) | `#super[1]` |

**Approach:** Use `markdown-it-py` to parse markdown into an AST, then walk the tree to emit Typst markup. This is more reliable than regex-based string replacement and handles nested formatting, escaped characters, and edge cases correctly.

#### Typst Template

A `.typ` file defining the document structure and styling:

```
Cover page
  - Course title (large, centered, serif)
  - "Generated by agent-learn" subtitle
  - Date

Table of contents (auto-generated)

For each section:
  - Section heading (starts on new page)
  - Section content (converted from markdown)
  - Inline citation superscripts [1], [2], etc.

References page (final page)
  - Numbered list of all unique citations across all sections
  - Each entry: [N] Source title. URL.
```

**Typography:**
- Font: New Computer Modern (serif, ships with Typst) or similar serif
- Body text: 11pt, 1.5 line spacing
- Margins: 2.5cm top/bottom, 3cm left/right
- Page numbers: bottom center
- Section headings: 18pt bold
- Cover title: 28pt bold, centered

#### Citation Aggregation

Each section has its own citations array with local numbering ([1], [2], etc.). For the PDF references section:

1. Collect all citations from all sections
2. Deduplicate by `source_url`
3. Assign global sequential numbers
4. Remap inline citation references in each section's content to use global numbers
5. Render the references list at the end

This means `[1]` in section 2 and `[1]` in section 4 might become `[3]` and `[7]` globally if they reference different sources.

### Frontend

#### Export Button

Add a download button to the learning page header (top of page, alongside existing navigation).

**Component:** A simple button in the existing learning page layout.

```
[Download icon] Export PDF
```

**Behavior:**
1. On click, fetch `/api/courses/{id}/export/pdf` with auth header
2. Show a loading spinner on the button while generating (PDF compilation may take 1-3 seconds)
3. On success, trigger browser file download
4. On error, show a toast notification with the error message
5. Button is only visible when course status is `completed`

**Placement:** Top of the learning page, in the header bar near the course title.

### Dependencies

#### Backend
- `typst` (Python package, `pip install typst`) — Typst compiler with Python bindings. Ships pre-compiled Rust binary, zero system dependencies.
- `markdown-it-py` — already used in the project or commonly available. Parses markdown to AST.

#### Frontend
- No new dependencies. Uses existing `fetch` + `Blob` + `URL.createObjectURL` for download.

#### Docker
- No system-level dependencies needed. `typst` pip package includes everything.
- The Typst template file (`.typ`) is bundled with the backend image.

## Error Handling

| Scenario | Behavior |
|----------|----------|
| Course not completed | 400 with message "Course must be completed before export" |
| Section has no content | Skip section in PDF (shouldn't happen for completed courses) |
| Section has no citations | Render section without citation superscripts |
| Typst compilation fails | 500 with logged error, user sees "Export failed" toast |
| Very long course (10+ sections) | Still works — Typst compiles fast, no timeout concern |
