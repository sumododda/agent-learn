# Milestone 4 — Interactive Learning

**Date:** 2026-03-19
**Status:** Design approved
**Approach:** Chat-First (Approach A)

## Goal

Make agent-learn a learning product, not just content generation. Learners engage with material through an AI chat assistant and visual diagrams — no quizzes.

## Scope

Two features:

1. **AI Chat Assistant** — bottom-drawer conversational interface on the learn page, full course context awareness, persistent conversation memory per course, streaming responses, user-selectable model via OpenRouter
2. **Mermaid Diagrams** — rendered in both section content (generated by writer agent) and chat responses (generated by assistant)

## Data Model

### New: `chat_messages` table

| Column          | Type     | Notes                                    |
|-----------------|----------|------------------------------------------|
| id              | UUID     | PK                                       |
| course_id       | UUID     | FK -> courses                            |
| user_id         | Text     | From Clerk auth                          |
| role            | Text     | "user" or "assistant"                    |
| content         | Text     | Message content (markdown)               |
| model           | Text     | OpenRouter model ID that generated response (null for user messages) |
| section_context | Integer  | Which section the user was reading       |
| created_at      | DateTime |                                          |

**Index:** `(course_id, user_id, created_at)` for fast conversation retrieval.

No changes to existing tables. The assistant reads from existing `Section`, `Blackboard`, and `EvidenceCard` tables for context.

## Backend API

### `GET /api/chat/models`

Proxies `GET https://openrouter.ai/api/v1/models` with `OPENROUTER_API_KEY`. Filters to text->text chat-capable models. Returns slimmed response per model:

```json
{ "id": "string", "name": "string", "context_length": 8192, "pricing": { "prompt": "0.00003", "completion": "0.00006" } }
```

Filters by `architecture.output_modalities` containing `"text"` and `architecture.input_modalities` containing `"text"`. Cached in-memory for 5 minutes.

### `POST /api/courses/{course_id}/chat`

Streaming SSE endpoint.

**Request:**
```json
{
  "message": "string",
  "model": "anthropic/claude-sonnet-4",
  "section_context": 3
}
```

**Response:** `text/event-stream`
```
data: {"choices":[{"delta":{"content":"The"}}]}
data: {"choices":[{"delta":{"content":" key"}}]}
...
data: [DONE]
```

**Flow:**
1. Authenticate via Clerk JWT
2. Load conversation history for user + course
3. Assemble context: course outline + current section content + blackboard + evidence cards + conversation history
4. Stream POST to `https://openrouter.ai/api/v1/chat/completions` with `stream: true` and user-selected model
5. Proxy SSE response to frontend
6. After stream completes, persist both user message and assistant response to `chat_messages`
7. If OpenRouter stream errors mid-response, persist user message but discard partial assistant response (don't save incomplete answers)

### `GET /api/courses/{course_id}/chat`

Returns paginated message history. Each message includes: `id, role, content, model, section_context, created_at`.

**Query params:** `?limit=50&before={message_id}` — cursor-based pagination, newest first. Default limit 50.

## Context Assembly

System prompt assembled per request:

```
You are a learning assistant for a course on "{topic}".
The learner is currently reading Section {N}: "{title}".
{instructions if provided}

--- COURSE OUTLINE ---
{all section titles + summaries, ~500 tokens}

--- CURRENT SECTION CONTENT ---
{full markdown of current section, ~1-2k tokens}

--- KEY TERMS (Blackboard) ---
{glossary terms + definitions, concept ownership map}

--- EVIDENCE FOR THIS SECTION ---
{verified evidence cards: claim, source, confidence}

--- CONVERSATION HISTORY ---
{last N messages, truncated oldest-first to fit budget}
```

**Token budget:**
- System context (outline + section + blackboard + evidence): ~4-6k tokens
- Conversation history: fill remaining up to ~8k tokens, drop oldest first
- Budget adapts to selected model's `context_length` from OpenRouter models list

**Assistant behaviors (from system prompt):**
- Answer based on course content and evidence cards first
- Reference specific sections when relevant
- Generate Mermaid diagrams when visual explanation helps
- Stay on-topic for the course
- Use blackboard glossary for consistent terminology

## Frontend — Chat Drawer

### Collapsed State
Floating pill button pinned to bottom-right of learn page: "Ask AI" with current model name. Click to expand.

### Expanded State
- Bottom drawer taking ~40% viewport height
- Header: "Assistant" label + "reading section N" context + model selector chip + close button
- Messages area: user messages right-aligned (bubble), AI messages left-aligned (no background, just text), supports markdown + Mermaid rendering
- Input: rounded text field with embedded send arrow button
- Streaming: AI messages appear token-by-token as SSE chunks arrive

### Model Selector
- Dropdown overlay triggered from model chip in drawer header
- Search bar at top
- Category quick-filters: All / Fast / Smart / Cheap
- Each model shows: name, provider, context length, pricing
- Selected model sent per-request (can switch mid-conversation)
- Models fetched live from OpenRouter via `GET /api/chat/models`

### Conversation Memory
- Single thread per course, persists indefinitely
- On drawer open: loads history via `GET /api/courses/{id}/chat`
- Messages include which section user was reading (section_context)

## Mermaid Diagrams

### Shared Renderer
- `MermaidBlock` React component detects ` ```mermaid ` code blocks in markdown
- Renders as inline SVG with dark theme
- Used by both section content renderer and chat message renderer
- Lightweight: mermaid library ~200kb gzipped

### Writer Agent Change
Prompt addition to existing writer agent in `agent_service.py`:

> "Where a concept benefits from a visual — process flows, architecture, relationships, state transitions — include a Mermaid diagram using a ```mermaid code block. Prefer flowchart, sequence, or graph diagrams. Keep them simple (under 15 nodes)."

No new backend code or Trigger.dev tasks for this — just a prompt change.

## Decisions Log

| Decision | Choice | Reason |
|----------|--------|--------|
| No quizzes | User preference | AI assistant replaces quiz engagement model |
| Bottom drawer (not side panel) | User preference | Keeps focus on content, available on demand |
| Full course + persistent memory | User preference | Richest context for useful answers |
| OpenRouter only | User preference | No provider abstraction needed |
| Model selectable in chat | User preference | Switch models on the fly from drawer |
| Streaming from day one | User preference | Essential for conversational UX |
| Diagrams in content + chat | User preference | Both channels benefit from visuals |
| Single thread per course | User preference | One conversation, persists indefinitely |
