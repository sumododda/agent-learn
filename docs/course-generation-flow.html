Creates 5-10 sections ordered by dependency, each with 3-5 must-answer research questions and source policy

Step 3 — Per-Section Pipeline (Sequential)
🔄 For each section — Sequential within, iterates all sections
A
🔍
verify-section Task
Verifier agent checks each evidence card: Is claim specific? Does passage support it? Confidence reasonable?
POST /api/internal/verify-section maxAttempts: 2
▸ details
  • Reviews each evidence card for quality and accuracy
  • Marks cards: verified=true/false with verification_note
  • Coverage check: ≥1 verified card per must-answer question
  • If gaps found: needs_more_research=true
  • Targeted re-research with search_depth="advanced" (2 credits/query)
  • Returns: cards_verified, cards_total, needs_more_research, gaps[]
↓ if pass
B
✍️
write-section Task
Writer agent generates markdown content with [N] citation markers using verified evidence cards + blackboard context
POST /api/internal/write-section maxAttempts: 3 OpenRouter LLM
▸ details
  • Input: verified evidence cards, blackboard (glossary, concept_ownership, coverage_map)
  • Structure: Why This Matters → Main Explanation → Examples → Key Takeaways → What Comes Next
  • Mandatory: starts with ## Section Title
  • Uses [N] markers for all factual claims (1-indexed to evidence cards)
  • Blackboard-aware: doesn't re-define glossary terms, references prior sections
  • Target: 400-800 words per section
  • DB writes: Section.content + Section.citations
↓ if pass
C
🖊️
edit-section Task
Editor agent polishes content, ensures consistency, verifies citations, and generates blackboard updates for downstream sections
POST /api/internal/edit-section maxAttempts: 2 DB: Section, Blackboard
▸ details
  • Polish: terminology consistency, transitions, repetition removal
  • Citation verification: ensures all [N] markers map to valid cards
  • Generates blackboard updates: new_glossary_terms, new_concept_ownership, topics_covered, key_points_summary, new_sources
  • Blackboard merging: update_blackboard() preserves prior section data
  • DB writes: Section.content (edited) + Blackboard rows
  • Blackboard enables downstream sections to build on earlier content coherently
